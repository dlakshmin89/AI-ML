What is Hyperpersonalization ?
Hyperpersonalization means using data and AI to give each customer a very customized experience based on their behavior, preferences, and past actions — not just their name
Companies use hyperpersonalization in banking, e-commerce, and CRM systems to increase engagement and customer satisfaction by showing the right content at the right time.
AI systems behaving very human-like, Context-aware
Personalization uses basic information like name or location. Hyperpersonalization uses real-time behavior, history, and AI predictions to customize the experience dynamically.

What is oops?
OOPS = Object-Oriented Programming System
OOPS is a programming methodology that organizes code into objects and classes to improve reusability, security, and maintainability.
Encapsulation
	“Encapsulation means wrapping data and methods together inside a class and restricting direct access to the data.”
 It protects sensitive information.
Example:
In a bank account class, the balance variable is private. You can only access it using methods like deposit() or withdraw().
Benefit:
Improves security and data protection.
Abstraction
	“Abstraction means hiding complex implementation details and showing only essential functionality to the user.”
The user sees what the object does, not how it works internally.
Example:
When driving a car, you use the steering wheel and pedals without knowing how the engine works.
Benefit:
Reduces complexity and improves usability.
Inheritance
	“Inheritance allows one class to inherit properties and methods from another class.”
It promotes code reuse.
Example:
A ‘Car’ class can inherit from a ‘Vehicle’ class.
Car automatically gets common features like speed and start().
Benefit:
Reduces code duplication and improves maintainability.
Polymorphism
	“Polymorphism allows the same method name to behave differently depending on the object.”
One interface, multiple implementations.
Example:
A method called makeSound():
	Dog → Bark
	Cat → Meow
Same method name, different behavior.
Benefit:
Improves flexibility and scalability.

“Together, these four principles help build modular, secure, reusable, and scalable applications.”
Or 
Encapsulation protects data by restricting access. Abstraction hides internal complexity. Inheritance enables code reuse through parent-child relationships. Polymorphism allows the same method to behave differently across objects
